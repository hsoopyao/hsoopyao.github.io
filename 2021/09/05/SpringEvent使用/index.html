🚀SpringEvent使用

本文最后更新于：2021年9月17日 上午

前言

场景：

  • 每个直播间有一个专属的会话Id(session Id)，当开始直播的时候会给在线的用户广播会话Id（这里我们使用的是websocket）
  • 这时就有一个问题，当时离线的用户登录的时候，就无法即时获得直播间的会话Id，我们应该在该用户上线的时候发送对应的会话Id给用户**(监听到用户登录连接时，给对应用户发送会话id，发送就用到websocket)**

而监听机制汉叔就说可以用到SpringEvent（事件监听机制）

之前没有听说过SpringEvent，所以要学习一下以及学会使用的场景，学会举一反三。

SpringEvent

Spring 官方文档翻译如下 ：

ApplicationContext 通过 ApplicationEvent 类和 ApplicationListener 接口进行事件处理。 如果将实现 ApplicationListener 接口的 bean 注入到上下文中，则每次使用 ApplicationContext 发布 ApplicationEvent 时，都会通知该 bean。 本质上，这是标准的观察者设计模式。

自定义事件中，继承ApplicationEvent

其中有字段source

/**
    * Create a new {@code ApplicationEvent}.
    *
    * @param source the object on which the event initially occurred or with
    *               which the event is associated (never {@code null})
    * 事件最初发生的对象或与事件相关联的对象 永远都不能为null(可以理解为上下文)
    */
   public ConnectCommandEvent(Object source) {
       super(source);
   }

实践

步骤

  1. 自定义事件，继承ApplicationEvent
  2. 自定义监听器，通过注解@EventListener
  3. 编写业务，引入发布者ApplicationEventPublisher

使用场景

监听用户订阅事件，当有用户订阅时，输出对应的信息

代码实现

自定义事件

@Getter
public class SubscriptCommandEvent extends ApplicationEvent {

    private String username;

    private String subscribe;

    /**
     * Create a new {@code ApplicationEvent}.
     *
     * @param source the object on which the event initially occurred or with
     *               which the event is associated (never {@code null})
     */
    public SubscriptCommandEvent(Object source,String username,String subscribe) {
        super(source);
        this.username = username;
        this.subscribe = subscribe;
    }

    public static SubscriptCommandEvent getInstance(Object source,String username,String subscribe){
        return new SubscriptCommandEvent(source, username,subscribe);
    }
}

自定义监听器

@Sl4j
@Component
public class ListenerService{
    @EventListener
    public void listenConnect(SubscriptCommandEvent commandEvent){
        log.info("client {} subscript {} channel",commandEvent.getUsername(),commandEvent.getSubscribe());
    }
}

左边会有一个电台的标识

编写业务层

ApplicationEventPublisher是ApplicationContext的父接口之一。这接口的作用是：Interface that encapsulates event publication functionality.

　　功能就是发布事件，也就是把某个事件告诉的所有与这个事件相关的监听器。

@Sl4j
@Service
public class SubscriptService{
    
    //发布者
    private final ApplicationEventPublisher publisher;

    public CoupleService(ApplicationEventPublisher publisher) {
        this.publisher = publisher;
    }
    
    public void connection(String username,String subscript){
        //参数source是与事件相关联的对象，就是类本身this
        publisher.publishEvent(ConnectCommandEvent.getInstance(this,username,subscript));
    }
}

左边会有一个耳机的样式

测试

@RestController
public class TestController {

    private CoupleService coupleService;

    public TestController(CoupleService coupleService) {
        this.coupleService = coupleService;
    }

    @GetMapping(value = "/{username}")
    public void subscript(@PathVariable String username){
        coupleService.connection(username,"RunningMan");
    }
}
