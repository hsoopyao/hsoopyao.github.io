💡Java注解与反射

本文最后更新于：2021年9月18日 上午

1. 注解

1.1 什么是注解?

注解与注释只相差一个字。
注释是注明解释，是让阅读代码的人更加清晰代码的逻辑性，JVM不会执行注释的内容，不会影响程序的运行

注解(Annotation)是代码的特殊标记，程序可以读取注解，一般用于代替配置文件

1.2 内置注解

我们平时已经接触过注解了 例如：

  • @Override：重写

  • @SuppressWarnings：镇压警告

    • 参数：all、unchecked、deprecation…

  • @FunctionalInterface：函数式接口

  • @Deprecated：不推荐使用

1.3 元注解

1.3.1 什么是元注解?

元注解的作用是负责注解其他的注解(禁止套娃!!)

Java中定义了4个标准的meta-annotation类型，他们被用来提供对其他annotation类型作说明

  • @Target：用于描述注解的使用范围

  • @Retention：表示需要在什么级别保存该注解信息，用于描述注解的生命周期

    • SOURCE：注解只保留在源文件，当Java文件编译成class文件时，注解被丢弃

    • CLASS：注解被保留到class文件中，但虚拟机加载class文件时，注解被丢弃(默认)

    • RUNTIME：class文件被加载后，仍存在

  • @Document：说明该注解将被包含在javadoc中

  • @Inherited：说明子类可以继承父类中的注解

我们常用的一般有@Target与@Retention

1.3.2 定义一个注解

狂神说要照猫画虎~
先找一个元注解

@SuppressWarning

@Target({TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, LOCAL_VARIABLE})
@Retention(RetentionPolicy.SOURCE)
public @interface SuppressWarnings {
    String[] value();
}

接下来我们自定义一个注解

//自定义注解
@Target(ElementType.METHOD)//作用于方法上
@Retention(RetentionPolicy.RUNTIME)//该注解加载后仍存在,默认是CLASS
@interface MyAnnotation{
	String value();//属性，类似方法的定义
	int age() default 10;//我们可以设置默认值
	String[] arr() default {"xxx","xxx"};
}

注意事项：

  1. 其中每一个方法实际上是声明了一个配置参数

  2. 属性的数据类型只能是基本类型,字符串,枚举,Class,注解

  3. 属性我们可以通过default来设置默认值

  4. 只有一个参数时，一般参数名为value，value可以省略

使用该注解

//@MyAnnotation(value = "xx",age = 10) 使用于类上则会报错
public class Demo{
	@MyAnnotation(value = "xx",age = 10)//arr不写也可以，因为有默认值
	public static void test(){
		System.out.println("test()");
	}
}

到后期我们学到反射，我们可以通过反射机制获取注解的信息从而进行进一步操作！

2. 强大的反射

在学习反射之前，我们要先知道什么是类对象
在学习类对象之前，我们要分清楚类的对象和类对象的区别

2.1 什么是类的对象?

类的对象：基于某个类new出来的对象，也称为实例对象

public static void main(String[] args){
	Person zhangsan = new Person();
	zhangsan.setName("zhangsan");
}

其中，zhangsan就是一个实例对象，也就是类的对象

2.1 什么是类对象?

类对象：类加载的产物，封装了一个类的所有信息(类名、父类、接口、属性、方法、构造方法、注解)

每一个类加载到内存后都对应一个Class对象，每个类有且只有一个Class对象

正常实例化与反射的对比

2.2 获得类对象的三种方法

2.2.1 通过Class.forName()的方式

参数是全路径包名

package com.bean.reflection;
class Person{
	public String name;
	private int age;
	double salary;
	//省略get()/set()/toString()方法
}

public class Test01{
	public static void main(String[] args) throws Exception{
		//获取Class对象
		Class c1 = Class.forName("com.bean.reflection.Person");
		System.out.println(c1);
	}
}

结果为：class com.bean.reflection.Person

2.2.2 通过对象名.getClass()的方式

2.4 通过Class对象获取类的信息

我们知道Class对象里面包括了很多类的信息，我们可以通过反射获取信息

2.4.1 获取类的属性

package com.bean.reflection;
import java.lang.reflect.Field;

/**
 * @ClassName Test01
 * @Description TODO
 * @Author Bean
 * @Date 2021/2/7 22:30
 * @Version 1.0
 */

class Person{
    public String name;
    private int age;
    double salary;
	//省略get/set/toString()方法
}

public class Test01 {
    public static void main(String[] args) throws Exception {
        //获取类对象
        Class c1 = Class.forName("com.bean.reflection.Person");
        //获取类中所有公共属性
        Field[] fields = c1.getFields();
        for (Field field : fields) {
            System.out.println(field);//public java.lang.String com.bean.reflection.Person.name
        }
        //获取类中所有属性
        fields = c1.getDeclaredFields();
        for (Field field : fields) {
            System.out.println(field);
            /*
                public java.lang.String com.bean.reflection.Person.name
                private int com.bean.reflection.Person.age
                double com.bean.reflection.Person.salary
             */
        }

        //获取类中指定公有属性
        Field field = c1.getField("name");
        System.out.println(field);
        //getField():只能获取公共(public)属性
        //System.out.println(c1.getField("salary"));编译错误
        //抛出异常：java.lang.NoSuchFieldException: salary

        //获取类中指定属性
        field = c1.getDeclaredField("salary");
        System.out.println(field);//double com.bean.reflection.Person.salary
    }
}

小总结：

  • getFields():返回Field数组类型，仅获取类中所有public修饰的属性

  • getDeclareFields():返回Field数组类型，获取类中所有属性包括(public default protected private)

  • getField():返回Field类型，仅获取指定公有属性

  • getDeclareField()：返回Field类型，可以获取所有指定属性

2.4.2 获取类的方法

package com.bean.reflection;
import java.lang.reflect.Field;
import java.lang.reflect.Method;

/**
 * @ClassName Test01
 * @Description TODO
 * @Author Bean
 * @Date 2021/2/7 22:30
 * @Version 1.0
 */

class Person{
    public String name;
    private int age;
    double salary;

    //私有方法
    private void privateMethod(){
        System.out.println("private");
    }
}

public class Test01 {
    public static void main(String[] args) throws Exception {
        //获取类对象
        Class c1 = Class.forName("com.bean.reflection.Person");
        //获取类中所有public方法以及父类的public方法
        Method[] methods = c1.getMethods();
        for (Method method : methods) {
            System.out.println(method);
            /*
                public java.lang.String com.bean.reflection.Person.toString()
                public java.lang.String com.bean.reflection.Person.getName()
                public void com.bean.reflection.Person.setName(java.lang.String)
                public int com.bean.reflection.Person.getAge()
                public void com.bean.reflection.Person.setAge(int)
                public double com.bean.reflection.Person.getSalary()
                public void com.bean.reflection.Person.setSalary(double)
                public final void java.lang.Object.wait() throws java.lang.InterruptedException
                public final void java.lang.Object.wait(long,int) throws java.lang.InterruptedException
                public final native void java.lang.Object.wait(long) throws java.lang.InterruptedException
                public boolean java.lang.Object.equals(java.lang.Object)
                public native int java.lang.Object.hashCode()
                public final native java.lang.Class java.lang.Object.getClass()
                public final native void java.lang.Object.notify()
                public final native void java.lang.Object.notifyAll()
             */
        }
        System.out.println("/////////////////////");
        //获取本类所有方法
        methods = c1.getDeclaredMethods();
        for (Method method : methods) {
            System.out.println(method);
            /*
                public java.lang.String com.bean.reflection.Person.toString()
                public java.lang.String com.bean.reflection.Person.getName()
                public void com.bean.reflection.Person.setName(java.lang.String)
                private void com.bean.reflection.Person.privateMethod()
                public int com.bean.reflection.Person.getAge()
                public void com.bean.reflection.Person.setAge(int)
                public double com.bean.reflection.Person.getSalary()
                public void com.bean.reflection.Person.setSalary(double)
             */
        }

        //获取父类或本类public指定方法
        Method method = c1.getMethod("wait");
        System.out.println(method);//public final void java.lang.Object.wait() throws java.lang.InterruptedException
        method = c1.getMethod("getAge");
        System.out.println(method);//public int com.bean.reflection.Person.getAge()

        //获取本类指定方法 getDeclaredMethod("方法名称",返回值类型)
        method = c1.getDeclaredMethod("privateMethod");
        System.out.println(method);//private void com.bean.reflection.Person.privateMethod()
        method = c1.getDeclaredMethod("setName", String.class);
        System.out.println(method);//public void com.bean.reflection.Person.setName(java.lang.String)

    }
}

小总结：

  • getMethods():获取父类及本类的所有**公共(public)**方法

  • getMethod(“方法名”,返回值类型.class):获取父类及本类指定公共(public)方法

    • 为什么会有返回值类型?因为方法种有方法重载，同一个方法名有不同的参数列表，这样能准确找到指定方法！

  • getDeclareMethods():获取本类所有方法

  • getDeclareMthod(“方法名”,返回值类型.class):获取本类指定的方法

2.4.3 获取类中构造方法

package com.bean.reflection;
import java.lang.reflect.Constructor;
import java.lang.reflect.Field;
import java.lang.reflect.Method;

/**
 * @ClassName Test01
 * @Description TODO
 * @Author Bean
 * @Date 2021/2/7 22:30
 * @Version 1.0
 */

class Person{
    public String name;
    private int age;
    double salary;

    private Person(String name){
        this.name = name;
    }

    public Person(){}

    public Person(String name, int age, double salary) {
        this.name = name;
        this.age = age;
        this.salary = salary;
    }
}

public class Test01 {
    public static void main(String[] args) throws Exception {
        //获取类对象
        Class c1 = Class.forName("com.bean.reflection.Person");
        //获取类中公有构造方法
        Constructor[] constructors = c1.getConstructors();
        for (Constructor constructor : constructors) {
            System.out.println(constructor);
            /*
                public com.bean.reflection.Person()
                public com.bean.reflection.Person(java.lang.String,int,double)
             */
        }
        System.out.println("///////////");
        //获取类中所有构造方法
        constructors = c1.getDeclaredConstructors();
        for (Constructor constructor : constructors) {
            System.out.println(constructor);
            /*
                public com.bean.reflection.Person()
                private com.bean.reflection.Person(java.lang.String)
                public com.bean.reflection.Person(java.lang.String,int,double)
             */
        }

        //获取指定的公有构造方法 getConstructor(参数类型...)
        Constructor constructor = c1.getConstructor();
        System.out.println(constructor);//public com.bean.reflection.Person()
        constructor = c1.getConstructor(String.class,int.class,double.class);
        System.out.println(constructor);//public com.bean.reflection.Person(java.lang.String,int,double)

        //获取指定构造方法
        constructor = c1.getDeclaredConstructor(String.class);
        System.out.println(constructor);//private com.bean.reflection.Person(java.lang.String)
    }
}

小总结：

  • getConstructors():获得类中所有public构造方法

  • getDeclareConstructors():获取类中所有构造方法

  • getConstructor(参数类型…):获取类中指定public构造方法

  • getDeclareConstructor():获取类中所有指定构造方法

2.4.4 调用类的方法

package com.bean.reflection;
import java.lang.reflect.Constructor;
import java.lang.reflect.Field;
import java.lang.reflect.Method;

/**
 * @ClassName Test01
 * @Description TODO
 * @Author Bean
 * @Date 2021/2/7 22:30
 * @Version 1.0
 */
public class Test01 {
    public static void main(String[] args) throws Exception {
        //获取类对象
        Class c1 = Class.forName("com.bean.reflection.Person");
        //通过调用构造方法创建对象
        Constructor constructor = c1.getConstructor();//获取无参构造
        //newInstance():实例化 相当于new关键字
        Person p1 = (Person) constructor.newInstance();
        System.out.println(p1);//Person{name='null', age=0, salary=0.0}
        Person p2 = (Person)c1.newInstance();//本质上是调用无参构造
        System.out.println(p2);//Person{name='null', age=0, salary=0.0}
        //调用有参构造
        Constructor constructor1 = c1.getConstructor(String.class, int.class, double.class);
        Person zhangsan = (Person) constructor1.newInstance("zhangsan", 16, 12000);
        System.out.println(zhangsan);//Person{name='zhangsan', age=16, salary=12000.0}

        //调用普通方法
        Method eat = c1.getDeclaredMethod("eat", String.class);
        //invoke:唤醒
        //invoke(实例对象,传递的参数);
        eat.invoke(zhangsan,zhangsan.getName());//zhangsan is eating...

        Method setName = c1.getDeclaredMethod("setName", String.class);
        setName.invoke(p1,"lisi");

        Method getName = c1.getDeclaredMethod("getName");
        System.out.println(getName.invoke(p1));//lisi

		//获得私有属性同set()方法设置值
		Field age = c1.getDeclaredField("age");
		//设置权限,因为是私有的属性,不可以直接操作
        age.setAccessible(true);
        age.set(p1,10);
        System.out.println(p1.getAge());//10

		//调用静态方法
        Method staticMethod = c1.getMethod("staticMethod");
        staticMethod.invoke(null);//调用静态方法不需要对象
    }
}

class Person{
    public String name;
    private int age;
    double salary;

    public void eat(String name){
        System.out.println(name+" is eating...");
    }

    //私有方法
    private void privateMethod(){
        System.out.println("private");
    }

	//静态方法
	public static void staticMethod(){
		System.out.println("static method");
	}

    private Person(String name){
        this.name = name;
    }

    public Person(){}

    public Person(String name, int age, double salary) {
        this.name = name;
        this.age = age;
        this.salary = salary;
    }
}

小总结：

  • newInstance():实例化，相当于new关键字

    • 类对象.newInstance():相当于调用无参构造创建一个实例化对象

  • 调用有参构造和普通方法都需要传入一个实例对象 invoke(实例对象,值) / newInstance(实例对象,值)

  • 静态方法不需要传入对象

  • 操作私有属性和方法时，需要设置权限 setAccessible(true);才可以操作

2.4.5 设计一个通用方法：调用任何对象方法的方法

public class Demo{

	public static void invokeAny(Object obj,String methodName,Class<?>[] types,Object...args) throws Exception{
	/**
     * 调用任何对象方法的方法
     * @param obj 实例对象
     * @param methodName 需要调用的方法名
     * @param types 方法的返回类型的Class对象
     * @param args 需要传入的参数
     * @throws Exception
     */
    public static void invokeAny(Object obj,String methodName,Class<?>[] types,Object...args) throws Exception{
        //获取类对象
        Class c1 = obj.getClass();
        //通过Class类获取对应的方法名
        Method method = c1.getDeclaredMethod(methodName,types);
        //调用方法
        method.invoke(obj,args);
    }

	//测试
    public static void main(String[] args) throws Exception {
        Properties pro = new Properties();
        invokeAny(pro,"setProperty",new Class[]{String.class,String.class},"name","zhangsan");
        System.out.println(pro);//{name=zhangsan}
    }
}

2.5 类的加载

  • 加载：将Class文件字节码内容加载到内存中，并将这些静态数据转换成方法区的运行时数据结构，然后生成一个代表这个类的java.lang.Class对象
  • 连接：将Java类的二进制代码合并到JVM的运行状态之中的过程
    • 验证：确保记载的类的信息符合JVM规范，没有安全方面的问题
    • 准备：正式为类变量(staitc)分配内存并设置类变量默认初始值的阶段，这些内存都将在方法区中进行分配
    • 解析：虚拟机常量池内的符号引用(常量名)替换为直接引用(地址)的过程
  • 初始化：
    • 执行类构造器()方法的过程、类构造器()方法由编译期自动收集类中所有类变量的复制动作和静态代码块中的语句合并产生的。
    • 当初始化一个类的时候，如果发现其父类还没有进行初始化，则需要先出发其父类的初始化
    • 虚拟机会保证一个类的()方法在多线程环境中被正确加锁和同步
package com.bean.reflection;

import java.lang.reflect.Method;
import java.util.Properties;

/**

2.6 类加载器的作用

将class文件字节码内容加载到内存中，并将这些静态数据转换成放啊去的运行时数据结构，然后在堆中生成一个代表这个类的java.lang.Class对象，作为方法区中类数据的访问入口。

public static void main(String[] args) {
        //获取当前系统类的加载器
        ClassLoader systemClassLoader = ClassLoader.getSystemClassLoader();
        System.out.println(systemClassLoader);//sun.misc.Launcher$AppClassLoader@18b4aac2

        //获取系统类加载器的父类加载器
        ClassLoader parent = systemClassLoader.getParent();
        System.out.println(parent);//sun.misc.Launcher$ExtClassLoader@14ae5a5

        //获取扩展类加载器的父类加载器
        ClassLoader parent1 = parent.getParent();
        //根系统加载器是有c++编写的，装载核心类库
        System.out.println(parent1);//null
    }

2.6.1 双亲委派机制

当我们写了一个类，加载器会一级一级地往上检查是否已经存在相同的类，若相同则使用类中已经装载的类，回去使用我们自己定义的类；若没有发现则尝试加载用户写的类
