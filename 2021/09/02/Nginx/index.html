Nginx

Ngin是一个高性能的HTTP和反向代理服务器，最常见的用途是将其设置未代理服务器。我们学习还是从配置文件中开始！！

nginx.conf

从配置文件中可以发现

  • 全局块

    • 全局块是默认配置文件从开始到events块之间的一部分内容，主要设置一些影响Nginx服务器整体运行的配置指令，因此，这些指令的作用域是Nginx服务器全局。

  • event块

    • events块涉及的指令主要影响Nginx服务器与用户的网络连接。常用到的设置包括是否开启对多worker process下的网络连接进行序列化，是否允许同时接收多个网络连接，选取哪种事件驱动模型处理连接请求，每个worker process可以同时支持的最大连接数等。

  • upstream块 可以有块

  • http块

    • server块 可以有多个
    • location块 可以有多个

举一个完整栗子

user  nginx; #用户组
worker_processes  auto; #指定工作线程数 number|auto

error_log  /var/log/nginx/error.log notice; #指定错误日志的路径和日志级别
pid        /var/run/nginx.pid; #指定pid文件存放路径


events {
	#设置允许每一个worker process同时开启的最大连接数，当每个工作进程接受的连接数超过这个值时将不再接收连接
    worker_connections  1024;
}


http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream; #配置默认类型
    
	# log_format指令，用于定义日志格式，此指令只能在http块中进行配置
    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

	# 定义了上面的日志格式后，可以以下面的形式使用日志
    access_log  /var/log/nginx/access.log  main;

	#开启关闭sendfile方式传输文件
    sendfile        on;
    #tcp_nopush     on;
	
	#务器端对连接的保持时间
    keepalive_timeout  65;

    #gzip  on;

	#include指令，包含conf.d中以.conf后缀的配置文件，可以将server块的配置单独写一个conf文件，这样配置文件结构会比较清晰
    include /etc/nginx/conf.d/*.conf;
}

default.conf

server {
    listen       80; #ipv4监听端口
    listen  [::]:80; #ipv6监听端口
    server_name  localhost; #监听地址

	upstream name {
		server 127.0.0.1:8080;
		server 127.0.0.1:8081 backup; #热备
		#ip_hash 负载均衡规则，默认是轮询
	}

    #access_log  /var/log/nginx/host.access.log  main;

    location / { #请求的url过滤
        root   /usr/share/nginx/html; #静态资源路径
        index  index.html index.htm; #设置默认页
        #proxy_pass http://name #请求转向name定义的服务器列表
        #deny 拒绝的ip
        #allow 允许的ip
    }

    # redirect server error pages to the static page /50x.html
    # 错误页
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }

    # proxy the PHP scripts to Apache listening on 127.0.0.1:80
    #

    #    proxy_pass   http://127.0.0.1;
    #}

    # pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000

    # deny access to .htaccess files, if Apache's document root
    # concurs with nginx's one
    #

    #    deny  all;
    #}

Location

location指令的功能是用来匹配不同的url请求，进而对请求做不同的处理和响应

location语法

匹配URL类型，有四种参数可选，当然也可以不带参数。location [ = | ~ | ~* | ^~ ] uri { … }

  1. “=” 精确匹配，内容要同表达式完全一直才匹配成功

    location = /app/{
	...
}
#只匹配 http://localhost:8080/app/

  2. "~" 执行正则匹配，区分大小写

    location ~ /App/{
	...
}
#只匹配http://localhost:8080/App/
#http://localhost:8080/app/ [匹配失败]

  3. "~*" 执行正则匹配，不区分大小写

    location ~* /App/ {
	...
}
#http://localhost:8080/App/ [匹配成功]
#http://localhost:8080/app/ [匹配成功]

  4. "^~" 表示普通字符串匹配上以后不再进行正则匹配

    location ^~ /index/ {
	...
}
#http://localhost:8080/index/ [匹配成功]
#http://localhost:8080/index/test [匹配成功]
#http://localhost:8080/test/index [匹配失败]

  5. 不加任何规则时，默认是大小写敏感的，前缀匹配，相当于加了"~"与"^~"

    location /index/ {
	...
}
#http://localhost:8080/index [匹配成功]
#http://localhost:8080/Index [匹配失败]
#http://localhost:8080/index/index.page [匹配成功]
#http://localhost:8080/test/index [匹配失败]

  6. "@" nginx内部跳转

    location /index/ {
  error_page 404 @index_error;
}
location @index_error {
  .....
}
#以 /index/ 开头的请求，如果链接的状态为 404。则会匹配到 @index_error 这条规则上。

修饰符优先级

  1. =
  2. 无
  3. ^~
  4. ~或～*
  5. (无)

当然这是理论上的东西，还是需要实践才可以

实践

通过docker，nginx运行

先自行创建项目目录 /data/project/test

  1. 从dockerhub上拉取nginx镜像

    docker pull nginx #不写版本号默认是最新

  2. 在项目目录下编写docker-compose.yml文件

    version '3'
services: 
	nginx: 
		image: nginx
		container_name: nginx
		restart: always
		ports:
			- 8799:8799

  3. 运行镜像

    注意要在项目目录下执行docker-compose命令

    up：启动

    -d：后台执行

    docker-compose up -d

  4. 进入容器中找到要映射的数据卷(配置文件，日志文件等)

    docker exec -it nginx /bin/bash

  5. 拷贝容器中的文件到宿主机对应目录

    docker cp nginx:/etc/nginx/nginx.conf ./nginx/conf/nginx.conf
docker cp nginx:/etc/nginx/conf.d ./nginx/conf/conf.d

  6. 修改docker-compose.yml文件，添加数据卷映射

    version '3'
services: 
	nginx: 
		image: nginx
		container_name: nginx
		restart: always
		ports:
			- 80:80
		volumes: 
			- ./nginx/conf/nginx.conf:/etc/nginx/nginx.conf
			- ./nginx/conf/conf.d:/etc/nginx/conf.d
			- ./nginx/html:/usr/share/nginx/html
			- ./nginx/logs:/var/log/nginx

    volumes还有一种使用的方式：

    version '3'
services: 
	nginx: 
		image: nginx
		container_name: nginx
		restart: always
		ports:
			- 80:80
		volumes: 
			- ./nginx/conf/nginx.conf:/etc/nginx/nginx.conf
			- ./nginx/conf/conf.d:/etc/nginx/conf.d
			- ./nginx/html:/usr/share/nginx/html
			- cmsqs_nginx_log:/var/log/nginx
volumes: 
	cmsqs_nginx_log:

    这样是什么意思呢，我们通过命令docker volumes ls查询映射数据卷的列表

    $ docker volume ls
DRIVER    VOLUME NAME
local     cmsqs_cmsqs_nginx_log
$ docker volume inspect cmsqs_cmsqs_nginx_log
[
    {
        "CreatedAt": "2021-10-27T10:25:50Z",
        "Driver": "local",
        "Labels": {
            "com.docker.compose.project": "cmsqs",
            "com.docker.compose.version": "1.25.5",
            "com.docker.compose.volume": "cmsqs_nginx_log"
        },
        "Mountpoint": "/var/lib/docker/volumes/cmsqs_cmsqs_nginx_log/_data",
        "Name": "cmsqs_cmsqs_nginx_log",
        "Options": null,
        "Scope": "local"
    }
]

    发现映射到了"/var/lib/docker/volumes/cmsqs_cmsqs_nginx_log/_data"该路径

    两种方式的区别是前者可以自定义存放文件夹位置即名称，更加人性化一些；后者会自动生成一串很长的字符串，是不可能记得住的。

  7. 配置文件(default.conf是默认的配置文件)

    $ vim nginx/conf/conf.d/default.conf
server {
    listen       80;
    listen  [::]:80;
    server_name  localhost;

    #access_log  /var/log/nginx/host.access.log  main;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }

    #error_page  404              /404.html;

    # redirect server error pages to the static page /50x.html
    #
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }

    # proxy the PHP scripts to Apache listening on 127.0.0.1:80
    #
    #location ~ \.php$ {
    #    proxy_pass   http://127.0.0.1;
    #}

    # pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000
    #
    #location ~ \.php$ {
    #    root           html;
    #    fastcgi_pass   127.0.0.1:9000;
    #    fastcgi_index  index.php;
    #    fastcgi_param  SCRIPT_FILENAME  /scripts$fastcgi_script_name;
    #    include        fastcgi_params;
    #}

    # deny access to .htaccess files, if Apache's document root
    # concurs with nginx's one
    #
    #location ~ /\.ht {
    #    deny  all;
    #}
}

  8. 重启nginx

    修改配置文件就用 up 指令重启镜像，否则用 restart 指令重启

    docker-compose up -d

  9. 查看运行日志

    -f：自动滚屏

    docker-compose logs -f nginx

  10. 运行127.0.0.1:80
