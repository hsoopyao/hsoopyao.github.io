Nginx

Ngin是一个高性能的HTTP和反向代理服务器，最常见的用途是将其设置未代理服务器。我们学习还是从配置文件中开始！！

nginx.conf

从配置文件中可以发现

  • upstream块 可以有块
  • http块
    • server块 可以有多个
    • location块 可以有多个

举一个完整栗子

upstream name {
	server 服务名 weight=权重s
	keepalive 连接时长
}

#代理服务器
server{
	listen #代理服务器监听的端口
	server_name #表示监听到之后需要转到哪里去
	
	#匹配到的路径
	location /api/ {
		proxy_pass #代理的地址 相当于一个转发 即localhost:8080/api/这个请求会被转发到指定的proxy_pass的路径
	}
	
	location / {
		root #里面配置了root这时表示当匹配这个请求的路径时，将会在这个文件夹内寻找相应的文件
		index #当没有指定主页时，默认会选择这个指定的文件，它可以有多个，并按顺序来加载，如果第一个不存在，则找第二个，依此类推
	}
}

Location

location指令的功能是用来匹配不同的url请求，进而对请求做不同的处理和响应

location语法

匹配URL类型，有四种参数可选，当然也可以不带参数。location [ = | ~ | ~* | ^~ ] uri { … }

  1. “=” 精确匹配，内容要同表达式完全一直才匹配成功

    location = /app/{
	...
}
#只匹配 http://localhost:8080/app/

  2. "~" 执行正则匹配，区分大小写

    location ~ /App/{
	...
}
#只匹配http://localhost:8080/App/
#http://localhost:8080/app/ [匹配失败]

  3. "~*" 执行正则匹配，不区分大小写

    location ~* /App/ {
	...
}
#http://localhost:8080/App/ [匹配成功]
#http://localhost:8080/app/ [匹配成功]

  4. "^~" 表示普通字符串匹配上以后不再进行正则匹配

    location ^~ /index/ {
	...
}
#http://localhost:8080/index/ [匹配成功]
#http://localhost:8080/index/test [匹配成功]
#http://localhost:8080/test/index [匹配失败]

  5. 不加任何规则时，默认是大小写敏感的，前缀匹配，相当于加了"~"与"^~"

    location /index/ {
	...
}
#http://localhost:8080/index [匹配成功]
#http://localhost:8080/Index [匹配失败]
#http://localhost:8080/index/index.page [匹配成功]
#http://localhost:8080/test/index [匹配失败]

  6. "@" nginx内部跳转

    location /index/ {
  error_page 404 @index_error;
}
location @index_error {
  .....
}
#以 /index/ 开头的请求，如果链接的状态为 404。则会匹配到 @index_error 这条规则上。

修饰符优先级

  1. =
  2. 无
  3. ^~
  4. ~或～*
  5. (无)

当然这是理论上的东西，还是需要实践才可以
