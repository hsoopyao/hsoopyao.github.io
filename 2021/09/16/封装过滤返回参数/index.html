🎃封装过滤返回参数

本文最后更新于：2021年9月17日 上午

详情请点进来看，不给看概要 （略略略

其实主要想测试一下这个文章摘要 hhh

前言

  • 项目中遇到的问题是我们获取到了所有的字段，但是有些是不需要暴露出去的，所以我们需要过滤返回的参数，把别人需要的暴露出去，剩下无关紧要的字段过滤掉~
  • 上百度查了很久，发现都是通过VO一大推get/set来封装想要的参数(其实我自己写的项目也是这么干的…尴尬)
  • 从一开始用HashMap的remove来过滤，到用Hutool的BeanUtil，后来也有用汉叔说的MapStruct

那么接下来就介绍一下我的演变过程

不可缺的案发现场

常规方法-VO

这算是大众化的方法了吧

从我的毕设拷一段来看看

public LoginInfoVO userInfo(User user){
    //get role name by user's id
    String userId = user.getUserId();
    List<String> roleName = userRoleService.getRoleNameByIds(userId);
    Integer totalCollections = collectionsService.totalCollections(userId);
    log.info("client role name -> {}",roleName);
    return LoginInfoVO.builder().userId(user.getUserId())
        .username(user.getUsername())
        .gender(user.getGender())
        .avatar(user.getAvatar())
        .roleName(roleName)
        .level(user.getLevel())
        .identityId(user.getIdentityId())
        .status(user.getStatus())
        .collectionsNum(totalCollections)
        .build();
}

(看这一大堆的get/set…)

就是通过一个VO类写出你要展示出的字段(或者说是要暴露的字段)，然后通过查出的所有字段选择对应的字段set进去，看起来既不美观也很麻烦。。

粗暴方法-HashMap

这个方法是在写项目的时候想出来的。。

项目通过调用对应的sdk返回了json格式的字符串，我们通过强转的方法将String转成了HashMap

Map<String,Object> map = (Map<String, Object>) JSON.parse(result);

这就拿到了所有的字段，然后通过一个一个remove掉。。

...    
	Map<String,Object> map = (Map<String, Object>) JSON.parse(result);
    map.remove("字段");
    ...
return map;

嗯。。非常粗暴简单，以至于写的时候一度怀疑自己

还有一个问题就是万一要过滤的字段很多，那就要写无数个map.remove("字段")…想想都可怕噢！

如果用这种方法来修改我的毕设的代码就会变成以下这个样子

  1. 将user通过BeanUtil转化成Map
  2. 从Map中remove要过滤的字段
public Map userInfo(User user){
    //get role name by user's id
    String userId = user.getUserId();
    List<String> roleName = userRoleService.getRoleNameByIds(userId);
    Integer totalCollections = collectionsService.totalCollections(userId);
    log.info("client role name -> {}",roleName);
    Map<String, Object> userMap = BeanUtil.beanToMap(user);
    userMap.remove("不要的字段");
    return userMap;
}

巧妙方法

所谓巧妙方法就是通过一些工具类来实现啦

1. BeanUtil

通过Hutool这个小工具包来实现Bean属性注入

基于BeanUtil.fillBean方法Hutool还提供了Map对象键值对注入Bean，其方法有：

官方栗子，最为致命

BeanUtil.fillBeanWithMap 使用Map填充bean

HashMap<String, Object> map = CollUtil.newHashMap();
map.put("name", "Joe");
map.put("age", 12);
map.put("openId", "DFDFSDFWERWER");

SubPerson person = BeanUtil.fillBeanWithMap(map, new SubPerson(), false);

其中第三个参数表示是否忽略大小写

附上官方网站


照猫画虎

ResourceResponse类是自行创建的VO视图类

public List<ResourceResponse> common(String result){
    List<ResourceResponse> resources = new ArrayList<>();
    Map<String,Object> map = (Map<String, Object>) JSON.parse(result);
    Map<String,Object> data = (Map<String, Object>)map.get("data");
    for(Map<String,Object> list :(List<Map<String, Object>>)data.get("list")){
        resources.add(BeanUtil.fillBeanWithMap(list,new ResourceResponse(),false));
    }
    return resources;
}

如果类中还包含别的类的信息，只要类型一致，属性都能注入进去

如果用这种方法来修改我的毕设的代码就会变成以下这个样子

  1. 将user通过BeanUtil转化成Map
  2. 将Map通过BeanUtil注入对应的VO类中返回
public LoginInfoVO userInfo(User user){
    //get role name by user's id
    String userId = user.getUserId();
    List<String> roleName = userRoleService.getRoleNameByIds(userId);
    Integer totalCollections = collectionsService.totalCollections(userId);
    log.info("client role name -> {}",roleName);
    Map<String, Object> userMap = BeanUtil.beanToMap(user);
    LoginInfoVO loginInfoVO = BeanUtil.fillBeanWithMap(userMap, LoginInfoVO.builder().build(), false);
    return loginInfoVO;
}

2. MapStruct

这个是新知识，通过汉叔了解到的

MapStruct就是帮助我们节省那一大堆get/set方法，我们只需要编写一个接口，写清楚哪个属性注入对应的哪个属性即可，剩下的由MapStruct帮助我们生成

首先的首先要导入依赖！

<dependency>
    <groupId>org.mapstruct</groupId>
    <!-- jdk8以下就使用mapstruct -->
    <artifactId>mapstruct-jdk8</artifactId>
    <version>1.2.0.Final</version>
</dependency>

<dependency>
    <groupId>org.mapstruct</groupId>
    <artifactId>mapstruct-processor</artifactId>
    <version>1.2.0.Final</version>
</dependency>

话不多说 上才艺(bushi 上栗子

  1. 首先需要一个VO类

    public class OrgResponse {
    public String orgName;
    public String orgIndexCode;
    public String parentOrgIndexCode;
}

  2. 开始编写MapStruct的接口

    我这里是将HashMap作为参数来注入 这就要用到expression表达式来获取HashMap中对应的参数

    也可以类作为参数，对应的参数就是source

    target就是要注入的类的属性

    @Mapper(componentModel = “spring”) 就是交给spring容器来管理

    @Mappers就是映射规则

    @Mapper(componentModel = "spring")
public interface UtilStruct {

    /**
     * 转化
     * @param param 实体
     * @return OrgResponse类
     */
    @Mappings({
            @Mapping(expression = "java(param.get(\"orgName\"))",target = "orgName"),
            @Mapping(expression = "java(param.get(\"orgIndexCode\"))",target = "orgIndexCode"),
            @Mapping(expression = "java(param.get(\"parentOrgIndexCode\"))",target = "parentOrgIndexCode"),
    })
    OrgResponse convert(Map<String, String> param);

}

  3. 编译后会在target包中的generated-source包中的annotations中看到自动生成的UtilStructImpl

    可以看到它帮助我们写了get/set方法

    @Generated(
    value = "org.mapstruct.ap.MappingProcessor",
    date = "2021-09-16T11:21:54+0800",
    comments = "version: 1.2.0.Final, compiler: javac, environment: Java 1.8.0_131 (Oracle Corporation)"
)
@Component
public class UtilStructImpl implements UtilStruct {

    @Override
    public OrgResponse convert(Map<String, String> param) {
        if ( param == null ) {
            return null;
        }

        OrgResponse orgResponse = new OrgResponse();

        orgResponse.setOrgIndexCode( param.get("orgIndexCode") );
        orgResponse.setOrgName( param.get("orgName") );
        orgResponse.setParentOrgIndexCode( param.get("parentOrgIndexCode") );

        return orgResponse;
    }
}

  4. 最后通过注入UtilStruct，执行对应的方法进行转化即可

    public List<OrgResponse> common(String result){
    List<OrgResponse> convert = new ArrayList<>();
    Map<String,Object> map = (Map<String, Object>) JSON.parse(result);
    Map<String, Object> data = (Map<String, Object>) map.get("data");
    for (Map<String,String> listMap:(List<Map<String,String>>) data.get("list")){
        convert.add(utilStruct.convert(listMap));//主要看这里!!
    }
    return convert;
}

如果从毕设的栗子来看，就用类来作为参数进行讲解

MapStruct类

这里我把MaleVo类中的nameVo映射成男人的姓名，年龄映射成妻子的年龄

@Mapper(componentModel = "spring")
public interface UtilStruct { 
    @Mappings({
        	//如果两个类有一样的变量名则必须指明是哪个类的变量
            @Mapping(source = "male.name",target = "nameVo"),
        	//source也可以是wife.age
            @Mapping(source = "male.wife.age",target = "ageVo")
    })
    MaleVo convert(Male male,Wife wife);

}

测试结果

也可以将这个实体映射 只要类型一致即可进行属性注入

@Data
@AllArgsConstructor
@NoArgsConstructor
public class MaleVo {
    private String nameVo;

    private Integer ageVO;

    private Wife myWife;
}
@Mapper(componentModel = "spring")
public interface UtilStruct {

    UtilStruct INSTANCE = Mappers.getMapper(UtilStruct.class);

    @Mappings({
            @Mapping(source = "male.name",target = "nameVo"),
            @Mapping(source = "wife.age",target = "ageVO"),
            @Mapping(source = "wife",target = "myWife")
    })
    MaleVo convert(Male male,Wife wife);
}

也可以通过看到自定生成的Impl来看底层实现

@Generated(
    value = "org.mapstruct.ap.MappingProcessor",
    date = "2021-09-17T10:07:52+0800",
    comments = "version: 1.2.0.Final, compiler: javac, environment: Java 1.8.0_131 (Oracle Corporation)"
)
@Component
public class UtilStructImpl implements UtilStruct {

    @Override
    public MaleVo convert(Male male, Wife wife) {
        if ( male == null && wife == null ) {
            return null;
        }

        MaleVo maleVo = new MaleVo();

        if ( male != null ) {
            maleVo.setNameVo( male.getName() );
        }
        if ( wife != null ) {
            maleVo.setAgeVO( wife.getAge() );
            maleVo.setMyWife( wife );
        }

        return maleVo;
    }
}
