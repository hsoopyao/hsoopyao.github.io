apidoc使用

前言

  • 公司项目用到了apiDoc这个生成API文档的工具
  • 汉叔说对比Swagger来说比较灵活，如果修改一个返回值在对应的类中修改，执行一行命令即可更新

老规矩的案发现场：>>>图中链接

apiDoc快速上手

安装

通过npm命令进行下载安装

实践

找到一个现成项目或者新建一个项目

我们项目是在一个子模块中新建一个apidoc的文件夹，在里面写一个类叫xxxDoc，然后开始写注释

我看网上很多人都是在方法上面写apidoc的注释，用一个类来集中写比较好~

来，上栗子

让我们来一一解释一下其中的注释：

  1. @api

    @api {method} path [title]

    用来指明方法名、请求地址以及标题[标题是可选的]

  2. @apiGroup

    @apiGroup name

    将方法进行分组

  3. @apiVersion

    @apiVersion version

    用于指定文档块的版本号

  4. @apiDescription

    @apiDescription text

    用于描述文档块

  5. @apiParam

    @apiParam [(group)] [{type}] [field=defaultValue] [description]

    其中type可以指定变量大小 {String{..5}}最多5个字符的字符串 / {String{2..5}}表示2-5个字符串

    {type=allowedValues}指定变量的可选值 {number=1,2,3,99}number值可以是1.2.3.99四个值其中一个

  6. @apiParamExample

    @apiParamExample [{type}] [title]

    ​ example

    用于展示请求参数的样式

  7. @apiSuccess

    @apiSuccess [(group)] [{type}] field [descpription]

    请求成功后返回的参数

  8. @apiSuccessExample

    @apiSuccessExample [{type}] [title]

    ​ example

    和@apiParamExample差不多，用来展示栗子

  9. @apiHeader

    @apiHeader [(group)] [{type}] [field=defaultValue] [description]

    用于定义API-Header请求头

    可以提取成公用的，因为每个api请求都有请求头

  10. @apiHeaderExample

    @apiHeaderExample [{type}] [title]

    ​ example

    用于展示apiHeader的栗子

  11. @apiDefine

    @apiDefine name [title] [description]

    该注解用于定义一个内嵌的文档块，一般同@apiUse来搭配使用

    如果说@apiDefine是定义，那么@apiUse就是引用

以上列出来的都是项目上用到的，因此特地学习一下，还有一些还没列出来，可以根据apidoc的官网来学习更多

配置apidoc.json

每次导出接口文档都必须要让apidoc读取到apidoc.json文件(如果未添加配置文件，导出报错)，你可以在你项目的根目录下添加apidoc.json文件，这个文件主要包含一些项目的描述信息，比如标题、简短的描述、版本等，你也可以加入一些可选的配置项，比如页眉、页脚、模板等。

其中header.md/footer.md需要自行定义！
