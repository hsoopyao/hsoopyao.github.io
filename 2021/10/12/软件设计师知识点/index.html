计算机组成与体系结构

机器语言

一个数在计算机中的二进制表示形式，叫做这个数的机器数。机器数是带符号的，在计算机用机器数的最高位存放符号，正数为0，负数为1，计算机字长为8位

如：3转化为二进制则是 0000 0011 这个二进制就是机器数，是计算机看得懂的数字

原码、反码、补码、移码

对于一个数，计算机要使用一定的编码方式进行存储，原码、反码、补码是机器存储一个具体数字的编码方式

  • 原码：用第一位表示符号，其余位表示值（十进制转二进制）

    • [+1]原 = 0000 0001 其中最高位是符号位 这里是正数所以是0

      [-1]原 = 1000 0001 其中最高位是符号位 这是是负数所以是1

  • 反码：正数的反码是其本身，负数的反码是在其原码的基础上，符号位不变，其余取反即可

    • [+1] = [0000 0001]原 = [0111 1110]反

      [-1] = [1000 0001]原 = [1111 1110]反

  • 补码：正数的补码是其本身，负数的补码是在其反码的基础上+1

    • [+1] = [0000 0001]原 = [0111 1110]反 = [0111 1111]补

      [-1] = [1000 0001]原 = [1111 1110]反 = [1111 1111]补

  • 移码：不管正负数，只要将其补码的符号位取反即可

    • [+1] = [0000 0001]原 = [0111 1110]反 = [0111 1111]补 = [1111 1111]移

      [-1] = [1000 0001]原 = [1111 1110]反 = [1111 1111]补 = [0111 1111]移

正数三码（原码反码补码）合一

浮点数表示

N = 尾数*基数^指数

科学记数法是一种记数的方法。把一个数表示成a与10的n次幂相乘的形式（1≤|a|<10，a不为分数形式，n为整数），这种记数法叫做科学记数法。

浮点数的结构：

阶符± 阶码e 数符± 尾数m
阶码的符号 指数 尾数的符号 小数点后的数

浮点数实际上就是二进制的科学计数法，阶码指科学计数法中的指数，尾数指科学计数法中小数点之后的部分。

例如对于十进制的3.14×10²，“阶码”就是2，而“尾数”则是14。
同理对于二进制的1.00111010×10^1000^，“阶码”就是1000，而“尾数”则是00111010。

浮点数运算顺序：

  • 对阶
  • 尾数运算
  • 结果格式化

对阶：

1.25×10^6^ 0.000125×10^10^ (小数向大数看齐，尾数后移)

1.225×10^10^

特点：

  1. 一般尾数用补码，阶码用移码
  2. 阶码的数位决定数的表示范围，位数越多，表示的范围越大
  3. 尾数的位数决定数的有效精度，位数越多，有效精度越高
  4. 对阶时，小数向大数看齐
  5. 对阶时通过较小的尾数右移实现的

校验码

码距：任何一种编码都由许多码字构成，，任意两个码字之间最少变化的二进制位数就成为数据的码距

常考的校验码有：

  • 奇偶校验码
    • 奇校验：整个校验码(有效信息位和校验位)中的”1”的个数位奇数
    • 偶校验：整个校验码(有效信息位和校验位)中的”1”的个数位偶数
    • 特点：可检查1位(奇数位)的错误，不可纠错
  • 海明校验码
    • 海明码校验公式：2^R^ ≥　M + R + 1(R是校验位，M是数据位/信息位)
    • 特点：可检验错误，也可纠错
  • 循环冗余校验码（CRC）
    • 模2除法校验，通过生成多项式作为除数
    • 特点：可检验错误，不可纠错

  1. 已知多项式和原报文，求CRC编码，如：使用多项式G(x)=x^5 + x^4 + x +1,对报文10100110进行CRC编码，则编码后的报文是什么？ 11000

    步骤1：对报文10100110，在末尾添加所给多项式的最高次阶个0，如本题为x^5，则添加5个0，变为：1010011000000。

CISC与RISC

  • CISC：复杂指令集

    • 特点
      • 指令数量多
      • 指令频率差别大，多寻址方式
      • 使用微码。

  • RISC：精简指令集。

    • 特点
      • 指令数量少
      • 操作寄存器
      • 单周期，少寻址方式
      • 多通用寄存器
      • 硬布线逻辑控制，适用于流水线。

CPU组成（运算器和控制器）

✅ 处理机主要由处理器、存储器、总线组成，总线包括 数据总线、地址总线、控制总线

✅ s一条总线同一时刻仅允许一个设备发送，但允许多个设备接收(分时双工)

CPU全称是中央控制处理单元，其组成部分分为两大部分

  • 运算器
    • 逻辑算术单元（ALU）：数据的算术运算和逻辑运算
    • 算术累加寄存器（AC）：通用寄存器，为ALU提供一个工作区，用于暂存数据
    • 数据缓冲寄存器：写内存时，暂存指令或数据
    • 状态条件寄存器：存状态标志与控制标志（如进位、溢出等状态）存在争议，也可以是控制器的一部分！
  • 控制器
    • 程序计数器（AC）：用于存储下一条要执行的指令的地址
    • 指令寄存器（IR）：用于存储将要执行的指令（包括地址）
    • 指令译码器：对指令的操作码字进行分析解释
    • 时序部件：提供时序控制信号

  1. 以下属于CPU的算术逻辑单位的部件是()

    A. 程序计数器

    B. 加法器

    C. 指令寄存器

    D. 指令译码器

解析：

程序计数器、指令寄存器、指令译码器属于CPU中的控制器

  1. 计算机指令一般包括操作码和地址码两部分，为分析执行一条指令，其()

    A. 操作码应存入指令寄存器(IR)，地址码应存入程序计数器(PC)

    B. 操作码应存入程序计数器(PC)，地址码应存入指令寄存器(IR)

    C. 操作码和地址码都应存入指令寄存器(IR)

    D. 操作码和地址码都应存入程序计数器(PC)

解析：

程序计数器(PC)存放的是下一跳即将要执行的指令的地址

指令寄存器(IR)存放的是即将要执行的指令及其地址

寻址方式

  • 立即寻址方式
    • 特点：操作数直接放在指令中，速度快，灵活性差
  • 直接寻址方式
    • 特点：指令中存放的时操作数的地址
  • 间接寻址方式
    • 特点：指令中存放了一个地址，这个地址对应的内容时操作数的地址
  • 寄存器寻址方式
    • 特点：寄存器存放操作数
  • 寄存器间接寻址方式
    • 特点：寄存器内存放的是操作数的地址

层次化存储

计算机中的存储系统采用分级存储体系的理论依据是程序访问的局部性(时间局部性，空间局部性)

时间局部性：刚被访问的内容，立即又被访问

空间局部性：刚被访问的内容，临近的空间很快被访问

Cache

==Cache位于CPU与主存之间==

✅在计算机的存储系统体系中，Cache是访问速度最快的层次(若有寄存器，则寄存器最快)

✅使用Cache改善系统性能的依据是程序的局部性原理

主存与Cache之间的地 址映射由硬件直接完成，与操作系统和程序员无关

主存与Cache之间的地址映射有三种形式：

  • 直接相联映像（冲突率最高，电路简单）
    • 例如将主存中的每个区域的0号页只能存在Cache的0号页，冲突高
  • 全相联映像（冲突率最低，电路复杂）
    • 例如主存中的0号页可以存在Cache的任意位置，冲突低
  • 组相联映像（冲突率与电路复杂度是上面两者折中）
    • 例如主存以及Cache中的0-5号页组成一个组，每个组只可以存在Cache中对应的的组的位置

冲突率 ： 直接相联映像 > 组相联映像 > 全相联映像

电路复杂度： 全相联映像 > 组相联映像 > 直接相联映像

主存编址计算

✅存储单元个数 = 最大地址 - 最小地址 + 1

✅编址内容：

  • 按字编址：存储体的存储单元是字存储单元，即最小寻址单位是一个字
  • 按字节编址：存储体的存储单元是字节存储单元，即最小寻址单位是一个字节

✅总容量 = 存储单元个数 * 编址内容

根据存储器所要求的容量和选定的存储芯片的容量，就可以计算出所需的芯片的总数，即

​ ✅总片数 = 总容量 / 每片的容量

例题：

内存按字节编址，地址从A0000H到CFFFFH的内容，共有(B)字节，若用存储容量为64K×8bit的存储器芯片构成该内存空间，至少需要(B)片

A. 80 B. 96 C. 160 D. 192

A. 2 B. 3 C. 5 D. 8

解析：

  • 已知编址内容按字节编址，已知最大和最小地址，可以求出存储单位个数。存储单位个数 = 最大地址 + 1 - 最小地址 CFFFFH + 1 - A0000H = 30000H

  • 已知编址内容是8bit，将存储单位个数转化为十进制，再进行单位转换。总容量 = 存储单元个数 * 编址内容 (1K = 2^10^) 3×16^4^×8bit/2^10^B 即总容量为3×2^6^ = 192KB

  • 已知总容量和每片的容量即可求出总片数。总片数 = 总容量 / 每片的容量 3×2^6^KB / 64K×8bit = 3

流水线

✅流水线是指在程序执行时多条指令重叠进行操作的一种准并行处理实现技术。各种部件同时处理时针对不同指令而言的，它们可同时为多条指令的不同部分进行工作，以提高各部件的利用率和指令的平均执行速度。==流水线是准并行技术==

流水线周期位执行时间最长的一段

流水线指令运行时间的计算公式：

  • 理论公式(默认)：一条指令执行的时间 + （执行次数 - 1） 流水线周期(执行最长的时间)*

  • 实践公式：执行过程分解步数×流水线周期 + (执行次数 - 1) × 流水线周期

流水线的吞吐率是指再单位时间内流水线所完成的任务数量或输出的结果数量，

计算流水线吞吐率的最基本公式是：TP = 指令条数/流水线执行时间

流水线最大吞吐率：TPmax = 1/流水线周期

例题：

  1. 若指令流水线把一条指令分为取指、分析和执行三部分的时间分别是T(取指)=2ns，T(分析)=4ns,T(执行)=1ns，则100条指令全部执行完毕需(403)ns.

解析：

一条指令的执行时间：2 + 4 + 1 = 7ns

执行指令条数：100条

流水线执行最长的时间：4ns

则 **(2+4+1)+(100-1)4 = 403ns*

中断处理过程

  1. CPU无需等待也不必查询I/O状态

  2. 当I/O系统准备好以后，发出中断请求信号通知CPU

  3. CPU接到中断请求后，保存正在执行程序的程序(==保存现场==)，打断的程序当前位置即为==断点==

  4. (通过==中断向量表==) 传入I/O中的服务程序的执行，完成I/O系统的数据交换

  5. 返回被打断的程序继续执行(恢复现场)

系统可靠性

✅ 可用于计算软件可靠性的公式：MTTF / (1 + MTTF) 其中MTTF表示平均无故障时间

可靠性计算涉及到三种系统：

  1. 串联系统

    计算方法：串联要可靠，得所有得R都可靠，即R1×R2×R3…×Rn

  2. 并联系统

    计算方法：并联要可靠，只要有一个可靠就可以了，即R = 1 - (1 - R1) × (1 - R2) × (1 - R3)

  3. 冗余系统(既有串联也有并联)

    计算方法：串联就相乘，并联就相减，即 R × ( 1 - (1 - R)^3^ ) × (1 - (1 - R)^2^)

性能指标

性能相关概念

数据在1秒当中能够通过的次数称为主频，通过的所需要的时间称为时钟周期

吞吐量与吞吐率：某一个单位时间间隔内所完成的任务量

响应时间：提交请求到完成请求的时间(RT)

完成时间：事务执行或者指令执行完成的时间(TAT)

✅ MIPS = 指令条数 / (执行时间 × 10^6^) = 主频 / CPI = 主频 × IPC

CPI：平均每条指令的平均时钟周期个数

IPC：每周期运行指令条数

MIPC：百万条指令每秒

时间表示范围(K M G)单位转换：1K = 10^3^ 1M = 10^3^K = 10^6^ 1G = 10^3^M = 10^6^K = 10^9^

容量表示范围(K M G)单位转换：1K = 2^10^ 1M = 2^10^K = 2^20^ 1G = 2^10^M = 2^30^

例题：

某计算机系统的CPU主频为2.8GHz。某应用程序包括3类指令，各类指令的CPI(执行每条指令所需要的时钟周期数)及指令比例如下表所示。执行该应用程序时的平均CPI为(C)；运算速度用MIPS表示，约为(B)

指令A 指令B 指令C
比例 35% 45% 20%
CPI 4 2 6

A. 25 B. 3 C. 3.5 D. 4

A. 700 B. 800 C. 930 D. 1100

解析：

  • 某计算机系统的CPU主频为2.8GHz。表示计算机在1秒当中能够有2.8G次脉冲，时钟周期就是主频的倒数，1/2.8GHz
  • CPI：平均每条指令的平均时钟周期个数
    • 0.35×4+0.45×2+0.2×6= 3.5
  • MIPS = 指令条数 / (执行时间 × 10^6^) 执行时间为 时钟周期 × 每个周期执行指令的时间
    • 1M / (3.5 × 1 / 2.8G) = 1M / (3.5 × 1 / 2.8 × 10^3^M) = 800MIPS

多媒体计算问题

✅ 图像容量计算

条件 示例
知道像素，位数 每个像素为16位，图像为640*480像素，求容量：640 * 480 * 16 / 8 = 614400B
知道像素，色数 640 * 480 像素，256色的图像，求容量： 640 * 480 * log2(256) / 8 = 304200B

✅ 音频容量计算 容量 = 采样频率(Hz) * 量化 / 采样位数(位) * 声道数 / 8 (单位是b则不需要/8)

✅ 视频容量计算 容量 = 每帧图像容量(Byte) * 每秒帧数 * 时间 + 音频容量 * 时间

👉例题：

某数码相机内置128MB的存储空间，拍摄分辨率设定为1600 * 1200像素，颜色深度为24位，若不采用压缩存储技术，使用内部存储器最多可以存储(D)张照片

A. 12 B. 22 C. 13 D. 23

解析：

每张照片的容量 = 1600 * 1200 * 24 / 8 = 5760000B

转换为MB ：5760000 / 1024 / 1024 = 5.493MB

用相机内置容量除以每张照片的容量 ：128 / 5.493 = 23.3

死锁

🔢 如果一个进程在等待一件不可能发生的事情，则进程就死锁了

至少需要多少个资源才系统才不会发生死锁？

✅ 至少资源数 = (进程所需要的资源数 - 1) * 进程数 + 1

栗子：

👉 系统有5个进程：A,B,C,D,E。这5个进程都需要4个系统资源，如果系统至少有多少个资源，则不可能发生死锁

A B C D E
3 3 3 3 3

若每个进程分配需要的资源数少一个，就会多出一个资源数，这多出的资源数可以自由分配，就不会产生死锁问题

即 (4- 1) * 5 + 1 = 16个

产生死锁的四大问题

  1. 互斥
  2. 保持和等待
  3. 不剥夺
  4. 环路等待

解决死锁问题的算法：银行家算法

分配原则

🚩 当一个进程对资源的最大需求量不超过系统中的资源数时可以接纳该进程

🚩 进程可以分期请求资源，但请求的总数不能超过最大需求量

🚩 当系统现有的资源不能满足进程尚需资源数时，对进程的请求可以推迟分配，但总能使进程在有限时间里得到资源

👉 栗子：

假设系统中有三类互斥资源R1.R2.R3，可用资源分别说9.8.5，在TO时刻系统中有P1.P2.P3.P4.P5五个进程，这些进程对资源的最大需求量和已分配资源数如下所示，如果进程按照(B)序列执行，那么系统状态是安全的

A. P1-P2-P4-P5-P3 B. P2-P4-P5-P1-P3 C. P2-P1-P4-P5-P3 D. P4-P2-P4-P1-P3

解析：

  • 我们可以算出剩余资源分别是 2 1 0 以及还需分配资源数

  • 只能给P2分配，其余都不够资源分配。分配P2之后P2会返回已分配的资源数，因此分配后剩余的资源变成了 4 2 1

  • 接着再看剩下的还需资源数，发现可以分配给P4，分配后剩余资源变成了 5 4 1

  • 后面以此类推得到答案

存储管理

页式存储：将程序与内存均划分为同样大小的块，以页为单位将程序调入内存

✅ 逻辑地址 = 页号 + 页内地址

✅ 物理地址 = 页帧号(物理块号) + 页内地址

👉栗子：

页式存储系统中，每个页的大小为4KB，即页内地址用12位来表示(4k = 2^12^)

优点：利用率高，碎片小，分配及管理简单

缺点：增加了系统开销，可能产生抖动现象

段式存储：按用户作业中的自然段来划分逻辑空间，然后调入内存，段的长度可以不一样

优点：多道程序共享内存，各段程序修改互不影响

缺点：内存利用率低，内存碎片浪费大

段页式存储：段式和页式的综合体，先分段，再分页，1个程序有若干个段，每个段中可以有若干个页，每个页的大小相同，但每个段的大小不同**(较少考)**

优点：空间浪费小，存储共享容易，存储保护容易，能动态连接

缺点：由于管理软件的增加，复杂性和开销也随之增加，需要的硬件以及占用的内容也有所增加，使得执行速度大大下降

磁盘管理

磁头就好比留声机中的放在碟片上的那根磁头

✅ 存取时间 = 寻道时间 + 等待时间

寻道时间是指磁头移动到磁道所需要的时间，等待时间为等待读写的扇区转到磁头下方所用的时间

读取磁盘数据的时间应包括以下三个部分

  1. 找磁道的时间(寻道时间)
  2. 找块(扇区)的时间，即旋转延迟时间
  3. 传输时间

👉栗子：

某磁盘磁头从一个磁道移至另一个磁道需要10ms。文件在磁盘上非连续存放，逻辑上相邻数据块的平均移动距离为10个磁道，每块的旋转延迟时间及传输时间分别为100ms和2ms，则读取一个100块的文件需要(D)ms时间

A. 10200 B. 11000 C. 11200 D. 20200

解析：

  • 已知每块的旋转延迟时间及传输时间分别为100ms和2ms
  • 寻道时间：10 * 10 = 100
  • (10 *10 + 100 + 2) * 100 = 20200

磁盘读取数据(单缓存区与双缓冲区)

处理数据可以理解为三个步骤处理：

  1. 从磁盘读入到缓冲区
  2. 从缓冲区读入到(内存)用户区
  3. 处理(内存)用户区数据

如果是单缓存区，则以为步骤1和2都需要访问临界资源—缓冲区，所以需要合并成一个操作阶段(缓冲区同一时刻只允许一个进程访问)

如果是双缓冲区，可以实现读入到缓冲区2和从缓冲区1读入到用户区并发

可以构造流水线进行计算

👉例题：

假设磁盘块与缓冲区大小相同，每个盘块读入缓冲区的时间为16us，由缓冲区送至用户区的时间是5us，在用户区内系统对每块数据的处理时间为1us。若用户需要将大小为10个磁盘块的Doc1文件逐块从磁盘读入缓冲区，并送至用户区进行处理，那么采用单缓冲区需要花费的时间为(****)us；采用双缓冲区需要花费的时间为()us

A. 160 B. 161 C. 166 **D.**211

A. 160 B. 161 C. 166 D. 211

解析：

流水线建立时间为16+5+1 = 22us。

使用单缓冲区读入和读取不可同时操作，因此最长数据传输时间为16+5，即(16+5+1) + (10 - 1) * 21 = 211us

若使用双缓冲区，读入可以使用一个缓冲区，读取可以使用另一个缓冲区，因此最长数据传输时间为16，即(16+5+1) + (10 - 1) * 16 = 166us

ER模型

ER模型用于表示类与类之间的关系

联系转关系模式：

  • 1：1联系：可将联系合并至任意一端的实体关系模式中
  • 1：n联系：可将联系合并至n段实体关系模式中
  • m：n联系：联系必须单独转成关系模式
  • 三个以上实体间的一个多元联系

关系代数

  • 交(∩)：R ∩ S，即将R与S中的相同的信息提取
  • 并(∪) ：R ∪ S，即将R与S中的信息合并
  • 差(-)：R - S，即删除R中存在S的记录
  • 笛卡尔积(×)：R × S，即将R中的每一条记录都与S中的记录相匹配
  • 投影(π)：投影运算是从关系的垂直方向进行运算
  • 选择(σ)：选择运算是从关系的水平方向进行运算
  • 自然连接(⋈)：相当于去重操作

规范化理论

候选键：唯一表示元组，且无冗余，可以有多个

主键：在候选键中人选一个，有且只有一个

外键：其他关系的主键

候选键

✅ 图示法求候选键

  1. 将详细的函数依赖关系，用”有向图”的方式表示
  2. 找出入度为0的属性，并以该属性集合为起点，。尝试遍历有向图，若能正常遍历图中所有结点，则该属性即为关系模式的候选键
  3. 若入度为0的属性集不能遍历图中所有节点，则需要尝试性将一些中间结点(既有入度也有出度)并入入度为0的属性集中，直至该集合能遍历所有结点，集合为候选键

👉 例题：

关系R(A,B,C)满足下列函数依赖：F{ B -> C , B -> A, A -> BC}，关系R的候选关键字为(B)

A. AB B. A和B C. A和BC D. AC和AB

解析：

图中没有入度为0的属性，所以尝试将中间结点(该题的中间结点时A和B)看是否能遍历所有的结点

其中A选项与B选项不同的是：

  • AB说明AB一起能够遍历整个图
  • B选项说明的是A/B/A和B都可以遍历整个图

模式分解

无损联接分解：指将一个关系模式分解成若干个关系模式后，通过自然联接和投影等运算仍能还原到原来的关系模式

✅ 公式法：**(适用于两个分解关系模式)**

R1∩R2 -> (R1 - R2)

R1∩R2 -> (R2 - R1)

✅ 表格法

将关系模式用表格的形式画出来

👉例题：

设R = ABC，F = {A -> B}，则分解p1 = {R1(AB)，R2(AC)}与分解p2 = {R1(AB)，R3(BC)}是否都为无损分解？

解析：

对于p1 = {R1(AB)，R2(AC)}

  1. R1 ∩ R2 = A

  2. R1 - R2 = B

  3. R2 - R1 = C

即推出 A -> B 或 A -> C 所以是无损分解

对于p2 = {R1(AB)，R3(BC)}

  1. R1 ∩ R2 = B

  2. R1 - R2 = A

  3. R2 - R1 = C

即推出 B -> A 或 B -> C 所以是有损分解

表格法：

对于p1 = {R1(AB)，R2(AC)}

A B C
R1 √ √
R2 √ √

由题可知 A -> B，所以R2中的B也可以通过A推出来，当所有属性都打勾说明可以还原

范式

🔢 第一范式(1NF)：在关系模式中，当且仅当所有域只包含原子值，即每个属性都是不可再分的数据项，则称关系模式R是第一范式

🔢 第二范式(2NF)：当且仅当关系模式R是第一范式，且每一个非主属性完全依赖候选键(没有不完全依赖)时，则称关系模式R时第二范式

🔢 第三范式(3NF)：当且仅当关系模式R是第二范式(2NF)，且R中没有非主属性传递依赖于候选键时，则称关系模式R是第三范式

🔢 BC范式(BCNF)：设R是一个关系模式，F是它的依赖集，R属于BCNF当且仅当其F中每个依赖的决定因素必定包含R的某个候选码

子网划分

信息安全

对称加密技术

发送者和接收者使用同一个密钥

特点：

  • 加密强度不高，但效率高
  • 密钥分发困难

常见的对称密钥算法：DES/3DES(三重DES)/RS/IDEA算法

非对称加密技术

不一样的加密和解密的密钥，分为公钥和私钥(成对出现)，公钥加密，相应的私钥解密

加密过程：发送方A将消息发送给接受方B，消息应该用B的公钥进行加密，这样一来，只要B的私钥才可以解密

即用接收方的私钥加密，用接收方的公钥解密

特点：

  • 机密速度慢，但强度高

常见的非对称加密算法：RSA/ECC

数字签名

验证是本人签发的消息，证明消息的真实性

加密过程：发送方A将消息发送给接受方B，消息应用发送方的私钥进行加密(签名过程)，接收方应用发送方的公钥来解密验证是否是发送方发送的消息

即用发送方的私钥加密，用发送方的公钥解密验证

信息摘要

数字摘要：由单项散列函数加密成固定长度的散列值(不可逆)

常用的消息摘要算法：MD5/SHA等，市场上广泛使用的MD5,SHA等算法的散列值为别为128和160位，由于SHA通常采用的密钥长度较长，因此安全性高于MD5

网络安全

网络攻击分类主要分为被动攻击和主动攻击两大块

  • 被动攻击
    • 监听(保密性)
      • 消息内容获取
      • 业务流分析
  • 主动攻击
    • 中断(可用性)
    • 篡改(完整性)
    • 伪造(真实性)

DoS(拒绝服务)

不断向网站发送伪造的请求，当正常的请求发送时，服务器由于忙于处理伪造的请求而拒绝服务正常的请求

重放攻击

攻击者抓取向服务器发送的有效数据包，并利用此数据包不断地向服务器发送，导致服务器一直应答此数据包，从而崩溃。

业务流分析

通过长期监听被攻击者的数据流，从而分析出相关业务流，可以依此了解被攻击者的一些倾向，常见的广告推送就是建立在业务流分析基础上的

防火墙

防火墙是一种放置在网络边界上，用于保护内部网络安全的网络设备，主要由服务访问规则、验证工具、包过滤、应用网关4部分组成。通过对流经数据流进行分析和检查，可实现对数据包的过滤、保存用户访问网络记录和服务器代理功能。

包过滤型：优点速度快费用低，缺点维护困难，只能阻止少部分ip欺骗，不支持有效的用户认证
双宿网关：位于内外网之间，实现对内外和外网物理上的隔离
屏蔽主机：由包过滤和双宿网关组合形成，优点实现网络层和应用层安全，缺点堡垒主机可能被绕过
屏蔽子网：目前安全最高

**安全区(非军事区/DMZ)**：连接外网和内网，同时向内网和外网提供服务

木马和病毒

病毒：编制或者在计算机程序中插入的波坏计算机功能或者破坏数据，影响计算机使用并且能够自我复制的一组计算机指令或者程序代码

木马：计算机木马时一种后门程序，常被黑客用作控制远程计算机的工具
